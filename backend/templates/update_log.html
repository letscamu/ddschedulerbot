Version History
MVP 2.0.2 March 2026 - Interface Improvements & Bug Fixes
  • Removed standalone "Generate Schedule": Removed the confusing Generate Schedule button from the left nav — schedule generation now lives exclusively in the Planner Workflow.
  • One-click final generation: Collapsed the two-step "Continue to Generate Final" → "Generate Final Schedule" flow into a single click.
  • Schedule page read-only: Schedule page now shows only the published schedule — no generate/simulate buttons. Displays "No Schedule Published" with a link to Planner Workflow when empty.
  • Hot List / Special Requests UX: Added visual separation between the Hot List (include/exclude checkboxes) and Special Requests (approve/reject) sections to prevent confusion.
  • Custom schedule fixes: Fixed two KeyErrors ('custom' and 'label') that prevented generating final schedules from custom configurations.
  • Core Mapping API fix: Fixed DataLoader wiring bug that prevented uploaded Core Mapping Excel files from rendering on the Core Mapping page.
  • Feedback Dev status editable: Admin users can now update the "Dev" status column on the Flies and Swatters page via an inline dropdown.
MVP 2.0.1 March 2026 - Scheduler Fixes & Data Quality
  • CRTD filter fix: Correctly excludes CRTD orders; OSO core override now applied before scheduling.
  • WO number normalization: SDR and OSO work order numbers are now normalized before matching, eliminating silent merge mismatches.
  • BLAST schedule improvements: Op# and Current Op Description columns added; pre-blast delays from the Stators VSM are now applied at scheduling time.
  • WIP core pre-marking overhaul: Op 1300 orders schedule as priority 0 (On Blaster); ops 1340–1620 pre-mark cores as occupied with elapsed-time subtraction.
  • Unscheduled Orders tab: All schedule exports now include a tab listing orders that could not be scheduled, with reason codes.
  • Misc fixes: Rotor/bearing filters, WIP blast display, scheduler start date, Special Requests WO UX, published badge, and Windows Unicode error in feedback pipeline.
MVP 2.0.0 March 2026 - DynaBot: Full Rebrand & New Home
  • Rebranded to DynaBot: The scheduling app is now DynaBot, live at dynabot.biz. All EstradaBot references have been updated throughout the interface.
  • Update Log renamed to Flies and Swatters: Consistent with CAMU branding — bugs are flies, fixes are swatters.
  • Infrastructure migration complete: Moved to the ddschedulerbot GCP project under the CAMU organization. CI/CD runs fully automated through GitHub Actions with Workload Identity Federation.
MVP 1.11 March 2026 - Excel Hot List Selection & Stale Request Purge
  • Excel hot list selection in planner workflow: Hot list entries loaded from an Excel file are now shown in Step 4 of the planner workflow with individual checkboxes. Uncheck any entry to exclude it from the final schedule without removing it from the file.
  • Automatic purge of stale pending requests: Pending special requests that have not been reviewed within 14 days are now automatically expired. Unmatched Mode B placeholder requests retain a 28-day grace period. Approved, rejected, and published requests are never affected.
MVP 1.10.4 February 2026 - Hotfix: Schedule Generation Freeze
  • Fixed schedule generation freeze: When all orders shared the same priority tier (e.g., all "Normal"), the scheduler's Phase 2 fallback scan incorrectly skipped every order — causing an infinite loop. The scheduler now correctly limits the skip to orders already checked in Phase 1.
  • Added scheduling safety guard: A maximum consecutive empty-slot counter (500) prevents runaway loops if an edge case causes persistent bottleneck detection. The scheduler will stop gracefully and report remaining unscheduled orders.
MVP 1.10.3 February 2026 - Scheduling Gap Fix & Injection Bottleneck Detection
  • Fixed non-working day night shift bug: Core return estimates were inflated across weekends because early morning hours (12-5 AM) on non-working days were incorrectly blocked, even when they belonged to the previous day's night shift. This caused artificial gaps in the Monday schedule.
  • Redesigned blast scheduling algorithm: The scheduler now scans the entire order pool for each takt slot. If the highest-priority order's core is unavailable, it pulls from lower-priority orders to fill the slot instead of leaving a gap.
  • Added downstream injection bottleneck detection: Before scheduling a part into a blast slot, the system now checks if the injection machines (Desmas) can handle it when it arrives downstream. A slot is only skipped if all orders would cause an injection bottleneck.
MVP 1.10.2 February 2026 - Blast Timing & Dedup Fix
  • Fix: Schedule start time now begins at 5:30 AM (after handover) instead of 5:20 AM (during handover)
  • Fix: Deduplicate orders by WO# from Sales Order imports — SAP exports with multiple rows per WO no longer schedule duplicate parts
  • Safety: BLAST arrival scheduling now validates start slot is at a valid working time
MVP 1.10.1 February 2026 - Scheduler Accuracy Fix
  • Fix: Core return time estimate now uses phased break-sensitive/insensitive calculation matching actual simulation behavior — eliminates artificial clustering in schedule and simulation
  • Improvement: Added injection machine contention buffer to core availability estimates for more realistic scheduling flow
MVP 1.10.0 February 2026 - Access Control & Reorder
  • User Management (RBAC): Admin user management page with add, edit role, reset password, and disable/enable. Users are now persisted to GCS instead of environment variables only. New “operator” role for shop floor users. All users can change their own password via the navbar dropdown.
  • Core Mapping View: New read-only Core Mapping page with Part-to-Core Mapping and Core Inventory tabs. Mismatch detection highlights parts referencing cores not in inventory with a summary count bar.
  • Schedule Reorder (Drag & Drop): Planners can now manually reorder the schedule using drag-and-drop (SortableJS). Cross-BLAST movement is allowed. Reorder changes the actual BLAST sequence reflected in Excel exports. Custom ordering is persisted to GCS and a warning is shown if regenerating while a custom order is active.
  • Navigation Updates: Core Mapping link visible to admin/mfgeng/planner. User Management link visible to admin. Sequence column added to schedule table.
MVP 1.9.0 February 2026 - Simulation Power
  • Extended Simulation — 6-Day Weeks & Skeleton Shifts: Added per-day shift configuration supporting full and skeleton modes. Skeleton mode keeps all machines available but adjusts takt time to reflect reduced staffing.
  • 4th Standard Scenario — 6 Days x 12 Hours: New preset scenario card runs Mon–Sat with full 12-hour day + night shifts, giving planners a quick view of maximum capacity.
  • Custom Configuration Panel: Planners can now build a fully custom schedule by configuring each day (Mon–Sat) independently: working yes/no, full or skeleton mode, day/night/both shifts, and per-day takt time (1–120 min). The result appears as a selectable scenario card alongside the standard presets.
  • CURE/QUENCH Continuous on Skeleton Days: Continuous operations (CURE, QUENCH) run as full working days on all scheduled days regardless of skeleton/full mode — only takt time and active shifts change.
MVP 1.8.2 February 2026 - Planner Workflow Fix
  • Fix Planner Workflow Stuck at Step 4: When no special requests exist, the planner workflow had no way to advance past the "Review Requests" step. Added a "Continue to Generate Final" button that appears when the requests table is empty, allowing planners to proceed directly to final schedule generation.
MVP 1.8.1 February 2026 - Planner Bugfix
  • Fix Planner "Select This Mode": Resolved bug where selecting a scenario returned "No scenarios simulated" due to gunicorn running 2 workers with independent in-memory state. Switched to 1 worker + 8 threads so planner workflow state is shared across all requests.
